# Osk Post-1.0.0 Roadmap

**Date:** 2026-03-28  
**Status:** 1.0.0 Released  
**Target:** 1.1.0 and beyond

---

## Executive Summary

1.0.0 ships with a validated, supportable foundation. This roadmap captures the work that was intentionally deferred or remains pending resource availability.

---

## Phase 1: Deferred Validation (Immediate Priority)

These items were documented as limitations in 1.0.0 and should be completed as soon as resources are available.

### 1.1 Real-Device Sensor Validation

**Status:** Pending 5-10 Chromebooks  
**Effort:** 1 day  
**Owner:** Test Lead

**Work:**
- Execute `scripts/chromebook_sensor_validation.md` procedure
- Test with 5 Chromebooks minimum, 10 if available
- Measure battery drain (<15%/hour target)
- Validate WebRTC connection stability
- Document any hub-side issues discovered

**Deliverable:**
- `docs/release/2026-XX-XX-sensor-validation-real-report.md`
- Update `docs/release/1.0.0-release-notes.md` with real-device results

**Trigger:** When Chromebook lab is available

---

### 1.2 Ollama Semantic Synthesis Validation

**Status:** Pending Ollama installation  
**Effort:** 2 hours - 2 days (depending on results)  
**Owner:** ML/AI Lead

**Work:**
- Install Ollama: `ollama pull llama3.2:3b`
- Run `scripts/evaluate_semantic_synthesis.py`
- Validate >80% accuracy target
- Validate <3s latency target
- Tune prompt if accuracy <80%

**Deliverable:**
- Accuracy report in `docs/release/2026-XX-XX-ollama-validation-report.md`
- Prompt tuning PR if needed
- Update config defaults if Ollama proves superior

**Trigger:** When Ollama is installed and running
