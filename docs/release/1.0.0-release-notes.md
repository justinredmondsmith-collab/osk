# Osk 1.0.0 Release Notes

**Version:** 1.0.0  
**Date:** 2026-03-28  
**Status:** Release Candidate  

---

## Summary

Osk 1.0.0 is a local-first civilian situational awareness platform for coordinators running protest or event operations. This release provides a hardened, validated foundation with explicit documentation of capabilities and limitations.

---

## What's New

### Evidence Pipeline (Complete)
- **Audio artifacts** - Raw audio chunks written to encrypted evidence store during transcription
- **Frame artifacts** - JPEG frames captured during vision analysis
- **Evidence export** - ZIP bundles with SHA256 checksums and integrity verification
- **Structured storage** - `{operation_id}/{member_id}/{audio|frames|metadata}/{timestamp}.{ext}`

### Sensor Streaming (Synthetically Validated)
- **5 sensor capacity** - Hub validated at 5 concurrent sensors (2.2% CPU, 679 observations/minute)
- **10 sensor projection** - Synthetic testing indicates capacity for 10+ sensors within targets
- **Synthetic validation** - Automated validation script for load testing
- **Chromebook procedure** - Documented lab procedure for real-device validation

### Semantic Synthesis (Code Complete)
- **Ollama backend** - LLM-powered synthesis with context-aware classification
- **Heuristic fallback** - Automatic fallback when Ollama unavailable
- **Configurable** - `synthesis_backend: "heuristic" | "ollama"`
- **Distinguishes context** - "Police helping" (INFO) vs "Police charging" (WARNING)

### Operational Hardening
- **SIGTERM deadline fix** - Wipe command exits cleanly with extended deadline
- **`--fresh` flag** - `osk start --fresh` for clean operation starts
- **Sensor validation tools** - Synthetic and browser-based validation scripts

## Supported Environment

### Coordinator
- Linux laptop (tested on Fedora 41)
- Python 3.11+
- Docker/Podman for local services
- NetworkManager for hotspot (optional)

### Member Browsers
- **Chromium-class browsers only**
- Chrome, Edge, Brave on desktop/Android
- ChromeOS (Chromebooks)

**Not supported:** Firefox, Safari/iOS, generic "mobile browsers"

## Configuration

```toml
# ~/.config/osk/config.toml

# Sensor streaming (validated synthetically at 5 sensors)
max_sensors = 10
audio_queue_size = 128
frame_queue_size = 64

# Synthesis (code validated, Ollama validation pending)

synthesis_model = "llama3.2:3b"

# Evidence storage (validated)
storage_backend = "luks"  # or "directory"
luks_volume_size_gb = 1
```

## Known Limitations

### Sensor Streaming
- **Synthetic validation only** - Real Chromebook validation pending
- **WebRTC not validated** - ICE/DTLS negotiation not tested with real browsers
- **Battery impact unknown** - Device battery drain not measured

### Semantic Synthesis
- **Ollama not validated** - Accuracy/latency tests pending Ollama availability
- **Heuristic is default** - Keyword matching remains default backend

### Browser Support
- **Chromium only** - Firefox, Safari explicitly not supported
- **PWA limitations** - Offline capability bounded to previously loaded pages
