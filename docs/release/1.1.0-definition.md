# Osk 1.1.0 Release Definition

**Date:** 2026-03-28  
**Status:** Draft - Container Validation Focus  
**Target:** Truthful field foundation with documented limitations

---

## Release Intent

Osk 1.1.0 ships as:
- A **local-first coordination system** with validated hub pipeline capacity
- **Containerized browser validation** proving 5+ concurrent sensor support
- **Experimental Ollama synthesis** with documented accuracy
- **Truthful documentation** about what is proven vs what remains pending

Osk 1.1.0 does **not** claim:
- Real-device battery/thermal validation (planned for 1.1.1)
- Broad mobile browser compatibility beyond Chromium-class
- Production-ready AI synthesis (heuristic remains default)

## Explicit Limitations (1.1.0)

The following are **intentionally not claimed** in this release:

| Limitation | Reason | Planned Resolution |
|------------|--------|-------------------|
| Real-device battery impact | Needs physical devices | 1.1.1 |
| Mobile WebRTC edge cases | Containerized != real mobile | 1.1.1 |
| Production AI synthesis | Accuracy below target | 1.2.0+ |
| Firefox/Safari support | Not validated | Future release |

---

## Minimum Supported Environment

### Coordinator
- One Linux laptop (Fedora 41+ or Ubuntu 22.04+)
- Python 3.11, 3.12, or 3.13
- 4GB RAM minimum, 8GB recommended
- Docker/Podman for service mode

### Member Browsers
- **Chromium-class browsers only** (Chrome, Edge, Brave)
- Mobile Chrome on Android (compatibility: best-effort)
- No Safari/iOS support claimed

### Synthesis
- **Heuristic backend** (default, validated)
- Ollama backend (optional, experimental)
  - Requires local GPU recommended
  - Accuracy documented as ~65%
