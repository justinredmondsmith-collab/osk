### 📋 After-Action Review System

Close operations with clarity, not confusion.

```bash
# Generate operation summary
$ osk aar generate --operation-id <uuid>

Operation Summary: March on Washington
======================================
Duration: 4h 23m
Members: 12 joined, 8 active at close
Findings: 47 total (3 critical, 12 warning, 32 info)
Tasks: 15 assigned, 12 completed, 3 pending
Media: 1.2 GB evidence collected

# Export complete evidence bundle
$ osk aar export --operation-id <uuid> --output march-aar.zip

Export Contents:
- operation-summary.json
- timeline-events.jsonl
- evidence-manifest.json (SHA-256 verified)
- media/ (audio, frames, metadata)
- closure-checklist.json

# Verify integrity
$ osk aar verify march-aar.zip
✅ Manifest signature valid
✅ All 1,247 files accounted for
✅ SHA-256 checksums verified
✅ Chain of custody intact
```

**AAR Components:**
- **Operation Summary** - High-level operation metrics
- **Timeline Events** - Chronological event reconstruction
- **Evidence Manifest** - SHA-256 integrity tracking
- **Closure Checklist** - Pending items and unresolved tasks
- **Integrity Verification** - Tamper-evident export validation

## Upgrade Guide

### From 1.0.0 / 1.1.0 / 1.2.0 / 1.3.0 / 1.4.0

```bash
# 1. Update package
pip install --upgrade osk

# 2. Verify new version
osk --version  # Should show 2.0.0

# 3. Run install readiness check
osk doctor --json

# 4. Review your support profile
# - If "supported-full": All features available
# - If "docker-managed": Use container deployment
# - If "supported-minimal": Some features limited
# - If "unsupported": Install missing dependencies

# 5. Test AAR workflow
osk start --fresh "Upgrade Test"
# ... run brief test operation ...
osk aar export --output test-aar.zip
osk aar verify test-aar.zip

# 6. Wipe test
osk drill wipe
osk wipe --yes
```

### Configuration Updates

```toml
# ~/.config/osk/config.toml

# New in 2.0.0: Security settings
[security]
operator_session_hours = 4
member_session_hours = 2
token_rotation_minutes = 30
max_concurrent_sessions = 5

# New in 2.0.0: AAR settings

# Existing settings (unchanged)
max_sensors = 10
synthesis_backend = "heuristic"
storage_backend = "luks"
```

## Breaking Changes

### Token Expiration
Session tokens now expire sooner:
- Operator sessions: 24h → 4h
- Member sessions: 12h → 2h

**Migration:** Operators may need to re-authenticate more frequently. Consider this a security improvement.

### CLI Changes
New commands:
- `osk doctor` - Install readiness check
- `osk aar generate` - Generate operation summary
- `osk aar export` - Export evidence bundle
- `osk aar verify` - Verify export integrity

Modified commands:
- `osk wipe` - Now includes verification logging

---

## Known Limitations

### Unchanged from 1.0.0
- **Sensor streaming:** Synthetic validation only; real-device battery/WebRTC pending
- **Semantic synthesis:** Code validated; Ollama accuracy testing ongoing
- **Browser support:** Chromium-class only; Firefox/Safari limits documented

### New in 2.0.0 (Documented)
- **Federation:** Multi-hub coordination not supported (3.x consideration)
- **Native apps:** Browser-only member experience
- **Forensic wipe:** OS/browser artifacts may remain; documented in `osk drill wipe`
