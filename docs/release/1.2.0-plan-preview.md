## 🔧 Workstreams

### 1. Tasking Model

Formalize coordinator-to-member tasking:

```
[Coordinator] ──assign──► [Member]
     │                        │
     │◄────ack/progress───────┤
     │                        │
     │◄──complete/timeout─────┤
```

**Components:**
- Task type definitions (confirmation, checkpoint, report, etc.)
- Assignment logic (capability-aware, freshness-aware)
- State machine: pending → assigned → acknowledged → in-progress → completed/timeout
- Retry and escalation rules

**Key Files to Touch:**
- `src/osk/operation.py` - Task lifecycle methods
- `src/osk/server.py` - Task API endpoints
- `src/osk/static/coordinator.js` - Task creation UI
- `src/osk/static/member.js` - Task view/acknowledgement UI

## 📊 Data Model Changes

### New: Task Entity

```python
@dataclass
class Task:
    id: uuid.UUID
    operation_id: uuid.UUID
    assigner_id: uuid.UUID  # Coordinator who created
    assignee_id: uuid.UUID  # Member assigned
    
    type: TaskType  # CONFIRMATION, CHECKPOINT, REPORT, etc.
    title: str
    description: str
    location: Optional[Location]  # Geo-target if relevant
    
    state: TaskState  # PENDING → ASSIGNED → ACKNOWLEDGED → IN_PROGRESS → COMPLETED/TIMEOUT
    created_at: datetime
    acknowledged_at: Optional[datetime]
    completed_at: Optional[datetime]
    timeout_at: datetime
    
    outcome: Optional[TaskOutcome]  # SUCCESS, FAILED, UNABLE, TIMEOUT
    notes: Optional[str]  # Member completion notes
```

### New: Route Confidence Model

```python
@dataclass  
class RouteAssessment:
    route_id: uuid.UUID
    origin: Location
    destination: Location
    
    confidence: float  # 0.0 - 1.0
    confidence_factors: List[ConfidenceFactor]
    
    status: RouteStatus  # CLEAR, CAUTION, BLOCKED, UNKNOWN
    last_assessed_at: datetime
    assessment_basis: List[EventReference]  # What events informed this
```

## 🔌 API Additions

### Task Management

```http
POST /api/operator/tasks
Authorization: Bearer {operator_token}
{
  "assignee_id": "uuid",
  "type": "CONFIRMATION",
  "title": "Check intersection",
  "description": "Confirm if Main St is passable",
  "timeout_minutes": 15
}
```

```http
GET /api/operator/tasks
Authorization: Bearer {operator_token}
Returns: List[Task]
```

```http
POST /api/member/tasks/{task_id}/acknowledge
Authorization: Bearer {member_token}
```

```http
POST /api/member/tasks/{task_id}/complete
Authorization: Bearer {member_token}
{
  "outcome": "SUCCESS|FAILED|UNABLE",
  "notes": "Optional completion notes"
}
```

### Route Confidence

```http
GET /api/operator/routes/assessment
Authorization: Bearer {operator_token}
{
  "origin": {"lat": 0.0, "lon": 0.0},
  "destination": {"lat": 0.0, "lon": 0.0}
}
Returns: RouteAssessment
```

```http
GET /api/operator/routes/gaps
Authorization: Bearer {operator_token}
Returns: List[Gap]  # Areas with insufficient confidence
```

## 🧪 Validation Plan

### Must Validate

1. **Task Flow End-to-End**
   - Coordinator creates task
   - Member receives notification
   - Member acknowledges
   - Member completes
   - Coordinator sees completion

2. **Reconnect Resilience**
   - Task assigned while member offline
   - Member reconnects → sees pending task
   - Task survives browser refresh

3. **Timeout Behavior**
   - Task times out after deadline
   - Coordinator notified
   - Member sees timeout

4. **Route Confidence Evaluation**
   - Compare route assessments against actual field conditions
   - Measure false positive/negative rates
   - Document confidence factor weights

### Evidence Required

- Task flow demo video or validation script
- Reconnect/resume verification logs
- Dashboard screenshots showing task views
- Audit artifact examples showing decision trail
