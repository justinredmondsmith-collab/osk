## Completed Work

### 1. Fixed: Wipe Shutdown Defect (SIGTERM Deadline)

**Problem:** The `osk wipe` command was returning exit code 1 even when the browser-side wipe succeeded. The shutdown sequence reused the same deadline for both graceful shutdown and SIGTERM fallback, leaving no time for the SIGTERM to work.

**Root Cause:** In `stop_hub()`, after sending SIGTERM at line 927, the loop continued using the original `deadline` which was already exhausted.

**Fix:** Extended the deadline after sending SIGTERM to give the process time to exit:

```python
# src/osk/hub.py line 929
sigterm_deadline = time.monotonic() + wait_seconds
while time.monotonic() < sigterm_deadline:  # Fresh deadline
```

**Files Changed:**
- `src/osk/hub.py` - Added SIGTERM-specific deadline
- `tests/test_hub.py` - Updated test to accommodate new deadline logic

### 3. Implemented: Real Evidence Writing in Intelligence Pipeline

**Problem:** The evidence store was opened during hub startup but nothing wrote to it, making `osk evidence export` fail with "No visible preserved evidence."

**Solution:** Added proper evidence writing throughout the pipeline:

1. **StorageManager** (`src/osk/storage.py`):
   - Added `write_evidence_artifact()` - Writes binary artifacts with structured paths
   - Added `write_evidence_metadata()` - Writes JSON metadata for each observation

2. **IntelligenceService** (`src/osk/intelligence_service.py`):
   - Added `storage` parameter
   - Added `_write_observation_to_evidence()` method
   - Integrated into `_persist_observation()` flow

3. **Hub** (`src/osk/hub.py`):
   - Pass `storage` to `IntelligenceService`

**Evidence Structure:**
```
~/.local/state/osk/evidence/
  {operation_id}/
    {member_id}/
      metadata/
        {observation_id}.json
      {artifact_type}/
        {timestamp}.{ext}
```

### 4. Implemented: `--fresh` Flag for Clean Operation Starts

**Problem:** `osk start` would resume stale active operations instead of creating fresh ones, interfering with clean validation runs.

**Solution:** Added `--fresh` flag that stops any active hub before starting:

```bash
# Start fresh operation (stops any active one first)
osk start --fresh "Osk 1.0.0 Release Validation"
```

**Files Changed:**
- `src/osk/cli.py` - Added `--fresh` argument to start parser
- `src/osk/hub.py` - `run_hub_sync()` handles `fresh` parameter
