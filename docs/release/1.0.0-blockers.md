# Osk 1.0.0 Blockers

This checklist is the canonical blocker list for the first `1.0.0` release.

Rules:

- A blocker stays open until there is written evidence in the repo.
- “Probably works” is not enough.
- Items may be narrowed or removed only by explicitly de-scoping them from the
  `1.0.0` definition.
- Scope expansion does not close blockers.

## Current Recommendation

The immediate next steps are:

1. Run the documented live-hub Chromium validation flow against a **clean**
   release-validation operation and retain its artifacts.
   Status: done, but it exposed command-level wipe/shutdown defects.
2. Fix or narrow the command-level wipe shutdown path so the same validated
   Chromium wipe flow exits cleanly without leaving stale hub state.
3. Fix or narrow the stale active-operation behavior that can force multiple
   stop cycles before a truly fresh validation run.
4. Produce at least one preserved-evidence bundle and complete
   `osk evidence export` plus `osk evidence verify`.
5. Decide whether inherited disconnected-member follow-up on resumed
   operations and post-wipe follow-up on timeout-path shutdowns are acceptable
   launch behavior or need explicit cleanup steps
   before `1.0.0`.
