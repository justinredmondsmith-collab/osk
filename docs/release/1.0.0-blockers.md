### NEW: Critical Capability Gaps (Must Resolve Before 1.0.0)

Based on capability assessment [`docs/release/1.0.0-capability-assessment.md`](./1.0.0-capability-assessment.md),
the following gaps must be resolved:

- [ ] **Evidence must contain actual media artifacts, not just metadata.**
  The evidence export currently contains JSON descriptions of observations
  but not the actual audio/video recordings. For 1.0.0, evidence export must
  include the source media files.
  
  Evidence needed: 
  - Evidence export ZIP contains audio/video files
  - Media files are playable/usable
  - Integrity verification passes
  - Implementation: `docs/release/1.0.0-gap-remediation-plan.md`

- [ ] **Sensor streaming must be validated at 5-10 concurrent sensors.**
  The audio/video capture code exists but has not been validated at the
  design scale. For 1.0.0, we must validate performance with multiple
  concurrent sensor streams.
  
  Evidence needed:
  - Performance report with 5+ concurrent sensors
  - Hub CPU/memory metrics under load
  - Member device battery impact assessment
  - Latency measurements (<5s end-to-end)
  - Implementation: `docs/release/1.0.0-gap-remediation-plan.md`

- [ ] **Synthesis must use semantic AI, not just keyword matching.**
  Current synthesis uses hardcoded keywords ("police", "medical"). For 1.0.0,
  synthesis must use local LLM (Ollama) for contextual understanding.
  
  Evidence needed:
  - Ollama synthesis backend operational
  - Accuracy validation vs keyword baseline
  - Latency <3s per synthesis
  - Configurable backend selection
  - Implementation: `docs/release/1.0.0-gap-remediation-plan.md`

- [ ] Close all launch-blocking failures or narrow scope until the matrix is
  green.
  Evidence needed: Re-run full validation matrix after gap remediation
